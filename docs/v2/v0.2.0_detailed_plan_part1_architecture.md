# OmniScope 0.2.0 详细开发方案 - Part 1: 架构设计

## 📋 执行摘要

**版本目标**: 将 OmniScope 从 Rust/Zig 专项工具升级为**业界首个通用跨语言FFI安全分析平台**

**核心指标**:
- ✅ **支持语言**: 7种（Python, Go, TinyGo, C/C++, C#, Java, Rust/Zig已有）
- ✅ **FP率**: 从 96.5% → **30%** (整体) / **5%** (boundary-only模式)
- ✅ **TP率**: 从 ~70% → **85%+** (FFI recall)
- ✅ **架构复用**: 100% 复用现有 MemoryGraph + SemanticTree + PassManager
- ✅ **开发周期**: 6-7周，分5个阶段渐进交付

---

## 🎯 核心设计哲学

### 设计原则: "薄适配层 + 厚核心引擎 + 零配置检测"

```
┌─────────────────────────────────────────────────────────────┐
│  Language Adapters (薄层 - 仅语义映射, ~200 LOC/语言)        │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐                │
│  │Python  │ │Go/CGO  │ │TinyGo  │ │C#/Java │  ...           │
│  │Adapter │ │Adapter │ │Adapter │ │Adapter │                │
│  └───┬────┘ └───┬────┘ └───┬────┘ └───┬────┘                │
├──────┼──────────┼──────────┼──────────┼──────────────────────┤
│      ▼          ▼          ▼          ▼                       │
│  Unified Semantic Layer (语言无关的语义抽象)                 │
│  • SemanticKind 扩展 (RAII, GC, RefCount, Manual)           │
│  • Ownership Model (Rust-style, C-style, GC-style)          │
│  • FFI Contract Database (配对规则 + 生命周期)               │
├───────────────────────────────────────────────────────────────┤
│      ▼                                                         │
│  Core Analysis Engine (复用现有 - 100%)                      │
│  ┌──────────┐  ┌──────────┐  ┌────────────────┐             │
│  │MemoryGraph│  │Semantic  │  │  PassManager   │             │
│  │(已有✅)   │  │Tree(已有)│  │  (已有✅)      │             │
│  └──────────┘  └──────────┘  └────────────────┘             │
└───────────────────────────────────────────────────────────────┘
```

**关键洞察**:
1. **LLVM IR 是通用语言** - 所有语言编译后都是 LLVM IR，差异仅在语义
2. **语义差异可枚举** - 7种语言的内存管理模型可归纳为4类：Manual, RAII, RefCount, GC
3. **现有架构已足够强大** - MemoryGraph 已支持跨函数追踪，只需扩展语义标注

## 🔗 与现有代码的集成点

### 集成点 A: 语义树扩展 (`src/semantics/semantic_tree.zig`)

**现状**: 已有 18 个 `SemanticKind` 变体，覆盖 Rust/Zig 的主要模式

**扩展策略**: 添加 16 个新变体，覆盖 Python/Go/C#/Java 的特有模式

```zig
// src/semantics/semantic_tree.zig (扩展)
pub const SemanticKind = enum(u8) {
    // ── 现有的保持不变 (18个) ──
    unknown, allocation, release, provenance,
    readonly_param, mutable_param, interior_mutability,
    heap_provenance, global_provenance,
    into_raw_transfer, file_operation, network_operation, process_operation,
    raii_drop_release, library_release,
    unsafe_transmute, uninit_memory_use, send_sync_violation,
    
    // ── 新增: Python (5个) ──
    python_refcount_inc,      // Py_INCREF
    python_refcount_dec,      // Py_DECREF
    python_borrowed_ref,      // PyList_GetItem
    python_owned_ref,         // PyBytes_FromString
    python_gil_protected,     // PyGILState_Ensure/Release
    
    // ── 新增: Go (4个) ──
    go_defer_cleanup,         // defer C.free(ptr)
    go_finalizer,             // runtime.SetFinalizer
    go_cgo_wrapper,           // _Cgo_* 包装函数
    go_runtime_alloc,         // runtime.mallocgc
    
    // ── 新增: C# (3个) ──
    csharp_safe_handle,       // SafeHandle.ReleaseHandle
    csharp_finalizer,         // ~Destructor()
    csharp_pinvoke_marshal,   // P/Invoke marshalling
    
    // ── 新增: Java (3个) ──
    java_local_ref,           // JNI LocalRef
    java_global_ref,          // JNI GlobalRef
    java_weak_ref,            // JNI WeakGlobalRef
    
    // ── 新增: C++ RAII 补充 (4个) ──
    cpp_unique_ptr,           // std::unique_ptr
    cpp_shared_ptr,           // std::shared_ptr
    cpp_destructor,           // ~ClassName()
    cpp_exception_path,       // try/catch 异常路径
};
```

**实现要点**:
- 每个新 `SemanticKind` 对应一个检测函数 (如 `detectPythonRefCount`)
- 复用现有的 `Resolution` 结构存储置信度和证据
- 通过 `pattern_id` 关联到语言适配器

### 集成点 B: 内存图增强 (`src/types/memory_graph_types.zig`)

**现状**: `AllocNode` 已有基础字段 (id, source_kind, freed 等)

**扩展策略**: 添加多语言生命周期追踪字段

```zig
// src/types/memory_graph_types.zig (扩展)
pub const AllocNode = struct {
    // ── 现有字段保持 ──
    id: u64,
    source_kind: SourceKind,
    freed: bool,
    family_id: ?FamilyId,
    // ... 其他现有字段 ...
    
    // ── 新增: 多语言生命周期追踪 ──
    
    /// 所有权模型类型
    ownership_model: OwnershipModel = .manual,
    
    /// RAII清理站点 (Rust drop_in_place, C++ destructor, C# SafeHandle)
    raii_cleanup_sites: std.ArrayList(u64) = undefined,
    has_raii_cleanup: bool = false,
    
    /// 引用计数追踪 (Python Py_INCREF/DECREF, C++ shared_ptr)
    refcount_ops: RefCountOps = .{},
    
    /// GC管理标记 (Java LocalRef, Go runtime.mallocgc)
    is_gc_managed: bool = false,
    gc_scope: ?GCScope = null,
    
    /// 延迟清理 (Go defer, C# using)
    has_deferred_cleanup: bool = false,
    defer_sites: std.ArrayList(u64) = undefined,
    
    /// 容器类型推断 (Box/Vec/String/unique_ptr/shared_ptr)
    container_type: ?ContainerType = null,
    
    /// FFI契约信息 (从契约数据库查询得到)
    ffi_contract: ?*const FFIContract = null,
    
    pub fn init(allocator: std.mem.Allocator, id: u64) AllocNode {
        return .{
            .id = id,
            .source_kind = .unknown,
            .freed = false,
            .family_id = null,
            .raii_cleanup_sites = std.ArrayList(u64).init(allocator),
            .defer_sites = std.ArrayList(u64).init(allocator),
        };
    }
};

/// 所有权模型枚举
pub const OwnershipModel = enum(u8) {
    manual,        // C/C++ malloc/free
    raii,          // Rust/C++ RAII
    refcount,      // Python/Objective-C
    gc,            // Java/Go
    hybrid,        // C# - GC + IDisposable
};

/// 引用计数操作记录
pub const RefCountOps = struct {
    increments: u32 = 0,
    decrements: u32 = 0,
    
    pub fn netCount(self: RefCountOps) i32 {
        return @as(i32, @intCast(self.increments)) - 
               @as(i32, @intCast(self.decrements));
    }
    
    pub fn isBalanced(self: RefCountOps) bool {
        return self.increments == self.decrements;
    }
};

/// GC作用域
pub const GCScope = enum(u8) {
    local,         // JNI LocalRef - 函数返回时自动释放
    global,        // JNI GlobalRef - 需手动DeleteGlobalRef
    weak,          // JNI WeakGlobalRef - 可能被GC回收
};

/// 容器类型
pub const ContainerType = enum(u8) {
    rust_box, rust_vec, rust_string, rust_rc, rust_arc,
    cpp_unique_ptr, cpp_shared_ptr, cpp_vector, cpp_string,
    python_list, python_dict,
    go_slice,
    csharp_list,
    java_arraylist,
};
```

**实现要点**:
- 所有新字段都有合理的默认值，不影响现有代码
- `init` 函数初始化动态数组
- 提供便捷方法 (如 `isBalanced`, `netCount`)
