##### FP 类别 B: Rust RAII 所有权系统 (18 issues, 占56%)

Rust 的 `Box<T>`, `String`, `Vec<T>`, `Rc<T>` 等类型在 **drop 时自动释放内存**。LLVM IR 层丢失了这种所有权语义。

| OMI-ID 范围 | 函数模式 | 类型 | 数量 | 典型示例 | 修复方案 |
|-------------|---------|------|------|---------|---------|
| OMI-002~015 | `core::ptr::drop_in_place` | cross_language_leak | 10 | `drop_in_place<$T>` 各种泛型实例化 | P1: 所有权感知分析 |
| OMI-016~019 | `alloc::vec::Vec::*` | cross_language_leak | 4 | `Vec::from_iter`, `Vec::finish_grow` | 同上 |
| OMI-020~021 | `alloc::boxed::box_new` | cross_language_leak | 2 | `Box::new_uninit` | 同上 |

**典型误报模式**:
```rust
// Rust 代码
fn process_data() -> Vec<u8> {
    let data = vec![0u8; 1024];  // 分配堆内存
    transform(&mut data);
    data  // 返回所有权给调用者，不是泄漏！
}
// OmniScope 看到: malloc 但没有 free → 误报为 orphan pointer
// 实际上: data 在调用者 drop 时自动释放
```

**LLVM IR 表现**:
```llvm
; Rust Vec::from_iter 编译后的 IR
define i8* @Vec_from_iter(...) {
entry:
  %0 = call i8* @__rust_alloc(i64 1024)  ; 分配
  ; ... 使用 ...
  ret i8* %0  ; 返回指针 ← OmniScope: "orphan pointer!"
}
; 调用者:
define void @caller() {
entry:
  %vec = call i8* @Vec_from_iter()
  ; ... 使用 ...
  call void @drop_in_place(i8* %vec)  ; 这里才真正释放！但不在同一个函数
}
```

**修复方案 (P1)**: 
实现 **所有权感知分析**，追踪 `drop_in_place` / `dealloc` 调用链：

```zig
// 新增: OwnershipAwareness.zig
pub fn trackRustDrop(graph: *MemoryGraph, inst: c.LLVMValueRef) !void {
    const callee = c.LLVMGetCalledValue(inst);
    const name = c.LLVMGetValueName(callee);
    
    // 识别 Rust drop intrinsic
    if (isRustDropIntrinsic(name)) {
        const ptr_arg = c.LLVMGetOperand(inst, 0);
        const ptr_val = @intFromPtr(ptr_arg);
        
        // 标记此指针有对应的 drop（不是泄漏）
        if (graph.findCanonicalAlloc(ptr_val)) |node| {
            node.has_drop_site = true;  // 新字段
            graph.recordDropSite(ptr_val, inst);
        }
    }
}

pub fn isLikelyRAIILeak(node: *const AllocNode) bool {
    // 如果存在对应的 drop site → 不是泄漏
    if (node.drop_sites.items.len > 0) return false;
    
    // 如果是 Box/String/Vec 分配 → 很可能有 RAII 清理
    if (isRustContainerType(node)) return true;  // 可能是FP
    
    return false;  // 无法判断
}
```

```zig
// 新增: RustInternalWhitelist.zig
const RUST_PANIC_FUNCTIONS = [_][]const u8{
    "panic_in_cleanup",
    "panic_fmt",
    "panic_bounds_check",
    "panic_cannot_unwind",
    "panic_count",  // is_zero_slow_path
    "begin_short_backtrace",  // sys::backtrace
    "catch_unwind.*cleanup",
};

const RUST_UNWIND_FUNCTIONS = [_][]const u8{
    "_ZN.*panicking.*",  // mangled std::panicking
    "_ZN.*core..result..Result.*catch_unwind",
};

pub fn isRustInternalFunction(func_name: []const u8) bool {
    for (RUST_PANIC_FUNCTIONS) |pattern| {
        if (std.mem.indexOf(u8, func_name, pattern) != null) return true;
    }
    for (RUST_UNWIND_FUNCTIONS) |pattern| {
        // 使用模糊匹配处理mangled名称
        if (fuzzyMatch(func_name, pattern)) return true;
    }
    return false;
}
```

- 集成到 `free_validation.zig` 的 `validateFreeWithMemoryGraph`

**3. 所有权感知分析** (增强现有 `MemoryGraph`)

- 扩展 `trackPointerFlow` 识别 RAII 模式
- 增强 `findCanonicalAlloc` 支持跨函数生命周期追踪

#### 1.1 Allocator Shim 识别器 [P0, 2天]

**目标**: 消除 19 个 bun_alloc FP (33% of total FP)

**实现文件**: `src/detectors/allocator_shim.zig` (新建)

**功能**:
```zig
/// Detects allocator vtable implementations to avoid false positives.
/// These functions follow the pattern: allocate-and-return or receive-and-free.
/// The actual lifecycle is managed by the caller via paired calls.
pub const AllocatorShimDetector = struct {
    /// Known allocator function patterns (mimalloc, rpmalloc, jemalloc, system)
    const PATTERNS = struct {
        const mimalloc = [_][]const u8{
            "mi_malloc", "mi_realloc", "mi_free",
            "mi_heap_malloc", "mi_heap_realloc", "mi_heap_free",
            "mi_malloc_aligned", "mi_realloc_aligned",
            "mi_stl_alloc", "mi_stl_delloc",
            "mi_malloc_small", "mi_calloc",
        };
        const system = [_][]const u8{
            "malloc", "calloc", "realloc", "free",
            "aligned_alloc", "posix_memalign",
        };
        const fallback = [_][]const u8{
            "fallback::",
            "__rust_alloc", "__rust_dealloc", "__rust_realloc",
        };
    };

    /// Check if a function is an allocator shim (should not report leak)
    pub fn isAllocatorShim(func_name: []const u8, context: AllocContext) bool {
        // Fast path: exact match against known patterns
        if (matchAnyPattern(func_name, PATTERNS.mimalloc)) return true;
        if (matchAnyPattern(func_name, PATTERNS.system)) {
            // System allocators need context: are they in a GlobalAlloc impl?
            return context.is_in_global_alloc_impl;
        }
        if (matchAnyPattern(func_name, PATTERNS.fallback)) return true;

        return false;
    }

    /// Enhanced: detect allocator by call-site context
    pub fn isAllocatorCallSite(inst: c.LLVMValueRef, graph: *MemoryGraph) bool {
        const caller = c.LLVMGetBasicBlockParent(
            c.LLVMGetInstructionParent(inst)
        );
        const caller_func = c.LLVMGetBasicBlockParent(caller);
        const caller_name = c.LLVMGetValueName(caller_func);

        // If the caller's name matches allocator trait impl pattern
        if (isGlobalAllocImpl(caller_name)) return true;

        return false;
    }
};
```

### Phase 2: 所有权感知分析 (Week 2-3) — 目标: 消除25个Rust RAII FP

#### 2.1 Rust Drop 追踪器 [P1, 3天]

**目标**: 理解 Rust 的 RAII 语义，追踪 `drop_in_place` 调用链

**核心洞察**:
```
Rust 代码:
  let x = Box::new(T { ... });  // __rust_alloc
  use(x);
} // x.drop_in_place → T::drop → __rust_free  (隐式!)

LLVM IR:
  define void @function() {
    %x = call i8* @__rust_alloc(...)     ; 分配
    call void @use_T(i8* %x)             ; 使用
    call void @drop_in_place(i8* %x)     ; 释放 (编译器插入!)
    ret void
  }
```

**关键**: OmniScope 需要识别 `drop_in_place` 是 **释放操作**，而非普通函数调用。

**实现**: 增强 MemoryGraph

```zig
// src/semantics/memory_graph.zig (增强)

/// New field on AllocNode to track RAII cleanup
pub const AllocNode = struct {
    // ... existing fields ...
    
    /// Set to true if a corresponding drop_in_place/dealloc was found
    has_raii_cleanup: bool = false,
    
    /// List of instructions that perform RAII cleanup for this allocation
    raii_cleanup_sites: std.ArrayList(u64),  // instruction addresses
};

/// New method: Track Rust drop intrinsics
pub fn trackRustDrop(graph: *MemoryGraph, inst: c.LLVMValueRef) !void {
    const called = c.LLVMGetCalledValue(inst);
    const name = std.mem.span(c.LLVMGetValueName(called));
    
    // Match: _ZN*$LT$T$GT$*drop_in_place* (mangled drop_in_place<T>)
    if (std.mem.indexOf(u8, name, "drop_in_place") != null or
        std.mem.indexOf(u8, name, "__rust_dealloc") != null)
    {
        const ptr_arg = c.LLVMGetOperand(inst, 0);
        const ptr_val = @intFromPtr(ptr_arg);
        
        if (graph.findCanonicalAlloc(ptr_val)) |node| {
            node.has_raii_cleanup = true;
            try node.raii_cleanup_sites.append(
                graph.allocator, 
                @intFromPtr(inst)
            );
            
            log.debug("RAII-DROP: Found cleanup for alloc 0x{x} at inst 0x{x}", .{
                node.id, @intFromPtr(inst)
            });
        }
    }
}

/// Query: Is this allocation likely cleaned up by RAII?
pub fn hasRAIICleanup(graph: *MemoryGraph, ptr_val: u64) bool {
    if (graph.findCanonicalAlloc(ptr_val)) |node| {
        return node.has_raii_cleanup;
    }
    return false;
}
```

```zig
pub const FFIContractDB = struct {
    contracts: std.AutoHashMap([]const u8, FFIRule),
    
    /// Query: Should this allocation be reported as potential leak?
    pub fn shouldReportLeak(db: *const FFIContractDB, alloc_func: []const u8) bool {
        if (db.contracts.get(alloc_func)) |rule| {
            switch (rule.management) {
                .gc => return false,  // GC-managed, not a leak
                .caller => return true, // Caller must free
                .callee => return false, // Callee manages
            }
        }
        return true;  // Unknown → conservative (report)
    }
    
    /// Query: Is this release function valid for the given allocation?
    pub fn isValidRelease(
        db: *const FFIContractDB,
        alloc_func: []const u8,
        release_func: []const u8,
    ) ?MatchResult {
        if (db.contracts.get(alloc_func)) |rule| {
            for (rule.releases) |valid_release| {
                if (std.mem.eql(u8, release_func, valid_release)) {
                    return .valid_pair;
                }
            }
            return .mismatch;  // Wrong release function!
        }
        return null;  // Unknown allocation
    }
    
    pub const MatchResult = enum {
        valid_pair,    // Correct pairing
        mismatch,      // Wrong release (potential bug!)
    };
    
    pub const FFIRule = struct {
        releases: [][]const u8,
        management: enum { gc, caller, callee },
        error_prone: bool = false,
    };
};
```

#### 3.2 跨函数生命周期追踪 (Scheme 4 from v0.1 plan) [P2, Week 4]

**目标**: 利用 MemoryGraph 的 call_ret edge 追踪完整的对象生命周期

```zig
/// Enhanced leak detection with cross-function tracking
pub fn isLeakedEnhanced(graph: *MemoryGraph, ptr_val: u64) ?LeakInfo {
    const node = graph.findCanonicalAlloc(ptr_val) orelse return null;
    if (node.freed) return null;  // Already freed
    
    // Step 1: Check if returned to caller (may be freed there)
    const ret_indices = graph.getCallRetsForPtr(ptr_val);
    for (ret_indices) |ret_idx| {
        const ret_edge = &graph.call_rets.items[ret_idx];
        
        // Does the caller eventually free this?
        if (graph.isEventuallyFreed(ret_edge.ret_ptr, max_depth=5)) {
            return null;  // Not leaked, freed in caller chain
        }
    }
    
    // Step 2: Check FFI contract DB
    if (node.alloc_func_name) |func| {
        if (!contract_db.shouldReportLeak(func)) {
            return null;  // GC/callee managed
        }
    }
    
    // Step 3: Check RAII cleanup
    if (node.has_raii_cleanup) {
        return null;  // Cleaned up by drop
    }
    
    // Really leaked!
    return LeakInfo{
        .node = node,
        .reason = determineLeakReason(node),
        .confidence = calculateConfidence(node),
    };
}
```

```zig
        "PyBytes_FromString",      # Returns new object (caller owns)
        "PyUnicode_FromString",     # Same
        "PyList_New",               # Returns new list
        "PyDict_New",               # Returns new dict
        "PyObject_CallObject",      # May return new object

    const BORROWING_FUNCTIONS = [_][]const u8{

        "PyTuple_GetItem",          # Borrows from tuple
```

```zig
    };
    
    /// Classify JNI reference type based on creation function
    pub fn classifyJNIRef(jni_func: []const u8) JNI_REF_TYPES {
        if (std.mem.indexOf(u8, jni_func, "NewGlobalRef") != null)
            return .global_ref;
        if (std.mem.indexOf(u8, jni_func, "NewLocalRef") != null)
            return .local_ref;
        return .local_ref;  // Default: most New* functions return LocalRef
    }
};
```

### 5.2 自动化测试脚本

```bash
#!/bin/bash
# tests/run_regression.sh

set -e

OMNISCOPE="./zig-build/bin/omniscope"
PASS=0
FAIL=0
TOTAL=0

for project in wasmtime bun python go cpp; do
    echo "▶ Testing $project..."
    
    for bc in tests/regression/$project/*.bc; do
        TOTAL=$((TOTAL + 1))
        
        # Run analysis
        output=$($OMNISCOPE "$bc" --json 2>/dev/null)
        
        # Compare with expected (allowing minor variance)
        if python3 tests/compare_results.py \
            "$output" \
            "tests/regression/$project/expected_issues.json"; then
            PASS=$((PASS + 1))
            echo "  ✅ $(basename $bc)"
        else
            FAIL=$((FAIL + 1))
            echo "  ❌ $(basename $bc)"
        fi
    done
done

echo ""
echo "Results: $PASS/$TOTAL passed, $FAIL failed"
exit $FAIL
```
