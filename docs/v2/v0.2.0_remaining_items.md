## 未完成项

### 1. FFI 契约数据库：缺少部分库

**来源**: Part 2, Part 8 发布清单

**现状**: 已有 8 个 builtin 库（OpenSSL, SQLite, Python, JNI, POSIX, glib, libuv, zlib），但计划中提到的以下库尚未实现：

- **cURL** — `curl_easy_*` 配对规则
- **BoringSSL** — 与 OpenSSL 合并或单独区分
- **JavaScriptCore** — GC 管理对象（`JSValueProtect`/`JSValueUnprotect`），在 bun_jsc 测试中大量出现
- **用户自定义契约** — `ContractSource::Custom` 枚举值已有但无加载机制

**优先级**: P1（JavaScriptCore 对 bun 项目精度提升最大）

---

### 2. 跨语言 Cross-Family Free 检测未达标

**来源**: graph_based_precision_plan.md 方案1, ffi_accuracy_development_plan.md P0.4

**现状**: `AllocatorShimDetector` 和 `FFIContractDB` 已实现，Resource Finding Reconciliation 已落地：
- TP=15, FP=12, FN=4, Precision=55.6%, Recall=78.9%
- Leak FP 从 5 降至 2，Leak Precision 83.3%
- Reconciliation 消除了同类误分类和重复计数 FP

**仍存缺口**:
- `malloc` -> C++ `operator delete` 跨 family 仍可能漏报
- `__rust_alloc` -> C `free` 路径需要更精确的 family 追踪
- 未知 family 默认 `C_HEAP` 导致误报

**优先级**: P0

---

### 3. Use-After-Free Through FFI 检测

**来源**: ffi_accuracy_development_plan.md P0.5

**现状**: `IssueGate` 和 `OwnershipSolver` 已有 UAF 检测基础，但以下场景仍为 FN：
- free 后指针传递给外部调用
- free 后指针作为间接调用参数
- 显式释放后 callback/userdata 使用

**需要**:
- 轻量级 alias propagation（bitcast, GEP, load/store, return forwarding）
- 外部调用和间接调用作为 FFI 边界 use 的识别
- codegraph 连接 release 和后续 FFI use

**优先级**: P0

---

### 4. Callback/Userdata Escape 检测

**来源**: ffi_accuracy_development_plan.md P0.6

**现状**: 无专门实现。以下模式未被检测：
- 栈/局部指针作为 callback userdata
- Rust/Go/Zig owned 指针被 C 侧存储后释放
- callback 注册无匹配的 unregister/revoke

**需要**:
- callee 名称启发式匹配（`register`, `set_callback`, `add_callback`, `subscribe`, `handler`）
- 函数指针 + 指针参数模式识别
- userdata 来源追踪（stack/local/borrowed/owned-by-managed-runtime）
- 缺少 unregister 配对检测

**优先级**: P0

---

### 5. FFI-focused Metric 分离

**来源**: ffi_accuracy_development_plan.md P0.1

**现状**: `accuracy_regression` 已实现分类指标（FFI / Resource / Leak / DoubleFree 分别统计），
并新增 `reconcile_subsumed` / `reconcile_deduped` 统计。仍缺：
- 每个 fixture 的 `boundary_kind`, `language_pair`, `resource_family` 元数据
- Runtime-noise FP 计数独立统计
- Known-FN recovery 计数

**需要**:
- 在 `ExpectedBug`/`ExpectedMiss` 结构中增加 FFI 元数据字段
- 单命令输出 TP/FP/FN + per-fixture 诊断

**优先级**: P0（度量基础）
